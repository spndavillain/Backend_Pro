Status Class

The status class provides a means to save and display status messages to the user. It is part of the Status Core Module in the application folder.

Note: This class is initialized automatically by the system so there is no need to do it manually.

How it Works

The Status class uses the CodeIgniter Session Class to save and keep track of status messages in your system. It does not do this by using the flashdata concept of the session class.. By not using the flashdata concept you don't have to worry about persisting your status messages accross multiple redirects. It will only show them when you want to.

Message Types

The status class can be used to display 5 different types of message.

  • notice - Used to show a hint or reminder to the user.
  • info - Used to prompt maybe a new site feature or a 'Did you know?' section.
  • success - Used to show a success message to the user after an action completed in full.
  • warning - Used to show a warning message. This should be used when execution can continue, but you just want to inform the user of maybe the concequence of their action.
  • error - Something has gone wrong, maybe a validation error maybe the site fell over.

Function Reference

$this->status->set('type', 'message')

The function sets a new status message with a given type. Below are some examples.

$this->status->set('info','A new product has now been released');
$this->status->set('success','Your new user has been created');

$this->status->get()

The function gets all status messages currently awaiting to be outputed. For example:

print $this->status->get();

If you only want to output a certain type of message you can do this by specifying the second optional parameter. The code below will print all error messages.

print $this->status->get('error');
