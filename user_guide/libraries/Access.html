Access Class

The Access Class provides a means to check whether a user has access to a controller, method or even a section of code. It is part of the Access Core Module in the application folder.

Note: This class is initialized automatically by the system so there is no need to do it manually.

Function Reference

$this->access->check('resource'')

The function is used to check if the current user has access to a specified resource. Where resource is the unique ID of the resource to check access for.

We can also perform a more detailed check by specifing an action to also check against. In the example below the user must not only be allowed access to the resource but also have permission to perform the action on the resource. Where action is the unique ID of the action.

$this->access->check('resource', 'action');

By default in the cases above if the user fails the access check, they will be redirected back along with an error message informing them they do not have permission. Now this is fine if you are guarding a controller but what if you want to only guard a few lines of code? We can do this by specifying a 3rd parameter.

if($this->access->check('resource', NULL, true))
{
        // Code to run if check passes
}

As you may have guessed, this 3rd parameter specifies whether the method should return its value or not. By default it is false and therefore a redirect is performed.
